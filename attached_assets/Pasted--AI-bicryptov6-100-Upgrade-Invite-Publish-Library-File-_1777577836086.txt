Found it. That screenshot is the smoking gun.

Your Custom Start Command is set to npm run start. That overrides everything in railway.json and nixpacks.toml. So Railway is running:

npm run start
→ node scripts/maintenance-off.js && pm2 start production.config.js --env production

pm2 start (not pm2-runtime) forks PM2 into the background and exits in under a second. The container's main process exits → container dies → healthcheck sees nothing → "service unavailable." Exactly what your log shows.

The fix (30 seconds)
In that Deploy settings panel:

Clear the "Custom Start Command" field completely so it's empty.
Click Save / Update (Railway will auto-redeploy).
Once the field is empty, Railway will fall back to your railway.json, which says bash railway-start.sh — that's the script that waits for MySQL, imports the schema, runs seeders, and uses pm2-runtime (foreground, won't exit).

Alternative if Railway's UI is being weird
If clearing the field doesn't stick, replace it with this exact value instead:

bash railway-start.sh

Same result.

About those red [CRON] lines
[CRON] ✗ Processing pending orders failed: Service temporarily unavailable

Don't panic — these are not deploy failures. Here's what they mean:

Bicrypto runs a background job every ~60 seconds called processPendingOrders. It checks for any pending exchange orders (spot trades, futures, etc.) and tries to settle them by calling the exchange API (Binance, KuCoin, etc.).

Since you haven't logged into the admin panel and configured exchange API keys yet, the call hits no upstream and returns "Service temporarily unavailable." The cron retries 3× and gives up for that minute, then tries again next minute. Totally normal for a fresh install. They'll stop the moment you either:

Configure exchange API keys in admin → settings → exchanges, or
Disable the spot-trading extension if you don't plan to use exchanges.
What to do right now
Open your *.up.railway.app URL. You should see the actual DeMourinho Crypto homepage now (not "Loading…", not "service unavailable").
Go to /login and sign in with:
